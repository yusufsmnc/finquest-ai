# Typography System — FinQuest AI

Bu dosya `ui-ux-pro-max` skill tarafından kullanılır.  
UI-Agent, tüm ekranlarda **tipografi kararlarını buradan alır**.

---

# 🧠 CORE RULE

## MAKSİMUM 2 FONT

Her UI tasarımında:

- 1 Heading font
- 1 Body font

❌ 3+ font = cognitive overload

---

# 🎮 TYPOGRAPHY STRATEGY SYSTEM

Tipografi seçimleri sadece estetik değil:

- decision clarity
- cognitive load reduction
- financial readability
- gamification emphasis

---

# 📊 1. MODERN FINTECH (DEFAULT)

### Inter + Inter

- Heading: Inter (600–700)
- Body: Inter (400–500)

### Kullanım
- dashboard
- scenario screen
- AI mentor

### Neden
- en yüksek okunabilirlik
- Flutter Material 3 native uyum
- nötr ve güvenli UI

---

# 💼 2. PRODUCTIVITY / DATA INTENSIVE

### Inter + JetBrains Mono

- Heading: Inter
- Numbers / Financial data: JetBrains Mono

### Kullanım
- portfolio
- risk analytics
- charts / numbers

### Neden
- sayısal verilerde maksimum netlik
- finans UI optimizasyonu

---

# 📐 TYPOGRAPHY SCALE (FLUTTER STANDARD)

## Body Text
- min: 14px
- ideal: 16px
- line height: 1.4 – 1.6

## Headings
- H1: 28 – 32
- H2: 22 – 26
- H3: 18 – 20

---

# 🎮 FINQUEST TYPOGRAPHY RULES

- Sayılar her zaman okunaklı ve net olmalı
- XP ve financial values mono hissi taşımalı
- AI feedback text daha soft okunmalı
- Scenario text en yüksek okunabilirlikte olmalı

---

# ⚙️ FLUTTER IMPLEMENTATION NOTE

- google_fonts kullanılabilir
- default fallback: Inter
- performans için max 2 font kuralı zorunlu

---

# ♿ ACCESSIBILITY RULES

- minimum 14px body text
- high contrast zorunlu
- okunabilirlik her şeyden önce gelir

---

# 🚫 STRICT RULES

- 3+ font yasak
- decorative font yasak (Comic Sans vb.)
- düşük contrast font yasak
- readability bozan font kombinasyonu yasak

---

# 🧠 FINAL PRINCIPLE

Typography is not visual decoration.

Typography = decision clarity + cognitive load control system
